# 2026-04-07 工作日志

## 画布缩放体验优化

### fitScale 计算修正
- **问题**：适应屏幕的高度估算值 `rect.height - 120` 不准确，导致大画布时 fitScale 偏大
- **修复**：改为精确扣除三部分：工具栏 52px + 容器 p-6 (48px) + 画布包裹层 p-6 (48px)

### 缩放后拖拽不灵敏
- **根因**：CSS `transform: scale()` 缩放画布后，`react-rnd` 内部的鼠标偏移量未被补偿。scale=0.5 时，鼠标移动 1px 在画布上只产生 0.5px 位移
- **修复**：给 `<Rnd>` 组件传入 `scale={scale}` 属性，让 react-rnd 内部自动补偿缩放坐标

### 缩放后拖放位置偏移
- **根因**：`addElementAt` 中 `clientX - r.left` 得到的是视觉像素距离，未转换到画布坐标系
- **修复**：除以当前 scale 得到画布原始坐标

### 缩放后元素松手后位置跳动
- **根因**：`updateElement` 和 `getBounds` 使用 `getBoundingClientRect()` 获取 bounds，但该方法返回的是 CSS scale 缩放后的视觉尺寸（如 scale=0.5 时 375px 画布返回 187.5px）。`applyBounds` 用这个缩小的 bounds 去 clamp 画布原始坐标的元素位置，导致位置被错误截断
- **修复**：`updateElement` 和 `getBounds` 都改为直接使用 `canvasSize`，始终以画布原始坐标系进行 clamp

### 文本元素不允许为空
- **问题**：文本内容清空后，画布上的文本框没有背景也没有边框，完全不可见，用户找不到
- **修复**：
  1. 侧栏 textarea 在 `onBlur` 时检查，如果文本为空则自动回退为 "输入文字"
  2. 画布渲染时空文本显示灰色占位文字「文本占位」兜底

### 适应屏幕按钮精确适配（最终方案：CSS zoom 替代 transform: scale）
- **根因**：`transform: scale()` 只改变视觉大小，不影响布局。`getBoundingClientRect()` 返回缩放后的视觉尺寸，导致 fitScale 计算不准确。且 `Rnd` 需要传 `scale` 属性做坐标补偿，容易出各种 bug
- **最终方案**：将整个缩放实现从 `CSS transform: scale()` 改为 `CSS zoom`
  - `zoom` 改变实际布局大小，`getBoundingClientRect()` 返回的是准确的布局尺寸
  - `handleFitScreen` 直接计算即可，不需要先 reset scale 再测量
  - `Rnd` 不需要传 `scale` 属性，内部计算自动正确
  - `onResizeStop` 中 `offsetWidth/Height` 是 zoom 后的值，需除以 scale 转回画布原始坐标
  - `addElementAt` 仍需除以 scale（zoom 同样缩放视觉）

### 缩放输入框移到画布右侧 → 最终改为右上角绝对定位
- 缩放输入框从画布上方移出，改为绝对定位在 main 区域的右上角（absolute right-6 top-6 z-10）
- 不随画布滚动，固定显示

### 文本元素对齐方式
- 在 PosterTextElement 中添加 `textAlign`（left/center/right）和 `verticalAlign`（top/center/bottom）
- ElementEditor 中添加水平/垂直对齐按钮组
- 画布渲染用 flexbox 实现对齐（justifyContent + alignItems + 内层 -webkit-box 做行数截断）
- GenerateTemplateClient Canvas 绘制中根据对齐方式计算 startX 和 startY
- PosterCanvasStatic 预览中也应用了对齐样式
- RenderPosterElement 类型中添加了 textAlign 和 verticalAlign 可选字段
- **垂直对齐修复**：改用 `flex-col` 外层 + 内层 `-webkit-box` 加 `margin-top/bottom: auto` 实现垂直对齐，避免内层被拉伸
- **缩放后拖拽修复**：给 `Rnd` 添加 `scale={scale}` 属性，补偿 CSS zoom 对 getBoundingClientRect 的影响
